# HybridSearchAgent_LangGraph

Learning/demo project for building a hybrid retrieval agent with LangGraph.

The folder demonstrates how to combine:

- Full-text search (SQLite FTS5)
- Vector search (Chroma + embeddings)
- Metadata-aware filtering
- Tool-driven agent orchestration with LangGraph/LangChain

All scripts in this folder are Learning/Demo status.

## Architecture Overview

1. Documents are loaded and chunked (mainly in the notebook).
2. Chunks are stored in two retrieval backends:
   - `fts.db` via `FTSStore`
   - `chroma_db` via Chroma vector store
3. `HybridRetriever` queries both backends and computes fused ranking.
4. `hybrid_search_tool` exposes hybrid retrieval to the LangGraph agent.
5. `hybrid_search_agent.py` runs an interactive loop with streaming events.

## Key Retrieval Behavior

- FTS mode options:
  - Keyword matching
  - Phrase matching
  - Prefix matching
  - Multi-mode fusion (weighted)
- Vector mode options:
  - Similarity
  - MMR (max marginal relevance)
- Final results are deduplicated and ranked by hybrid score.

## Configuration Flags

In `config.py`:

- `DEBUG`: Enable internal retriever debug prints
- `UPDATES`: Show LangGraph node state updates
- `DRAW`: Export graph PNG on startup
- `DEBUG_PRINT`: Enable helper debug output from utilities/tools
- `PRINT`: Control top-level console logging

## Suggested Learning Path

1. Start with `pydantic_models.py` to understand result/metadata schemas.
2. Read `fts_search.py` for FTS index and query behavior.
3. Read `hybrid_search.py` for score fusion and deduplication logic.
4. Run `hybrid_search_agent.py` for end-to-end interactive usage.
5. Explore `chroma_fts_metadata_chunks.ipynb` to understand ingestion pipeline details.

## Notes

- This repository is intentionally educational and experimentation-oriented.
- Existing DB artifacts (`fts.db`, `chroma_db/...`) are stateful; deleting them resets indexed state.
- The notebook currently has cells present but not executed in the current saved state.
